1. Scope of var, let, and const

JavaScript has three keywords for declaring variables: var, let, and const. Each has its own scope and usage rules.

`var`

  • Scope: Function scope; accessible within the function or globally if declared outside any function.
  • Hoisting: Variables are hoisted to the top of their scope, which can lead to unexpected behavior.
  • Reassignment and Redeclaration: Allows both reassignment and redeclaration within the same scope.

`let`

  • Scope: Block scope; confined to the block where it is declared.
  • Hoisting: Hoisted but inaccessible before its declaration.
  • Reassignment and Redeclaration: Allows reassignment but not redeclaration within the same scope.

`const`

  • Scope: Block scope.
  • Hoisting: Behaves like `let` in terms of hoisting.
  • Reassignment and Redeclaration: Does not allow reassignment or redeclaration within the same scope.

2. Use Cases of `null` and `undefined`

`null` and `undefined` are special values in JavaScript representing the absence of a value or a variable that hasn't been initialized.

`undefined`

  • Represents a variable that hasn't been assigned a value yet.
  • Used when declaring a variable but not initializing it.
  • Represents a missing parameter in a function call or an unassigned object property.

`null`

  • Represents the intentional absence of a value.
  • Commonly used to indicate a non-existent object or a cleared variable.
  • Useful for explicit checks where a value could be `null`.

Use Cases: `undefined` for uninitialized variables, and `null` for explicitly cleared or non-existent values.

3. What is a REST API?

REST (Representational State Transfer) is an architectural style for building networked applications, often used in web services. A REST API is an interface that follows the REST principles to interact with resources on a server.

Key Principles of REST

  • Statelessness: Each request must contain all the information needed to understand and process the request.
  • Resource-Based: REST APIs are organized around resources, identified by unique URLs.
  • HTTP Methods: Use standard HTTP methods like `GET`, `POST`, `PUT`, and `DELETE` to interact with resources.
  • Representation: Data is exchanged in formats like JSON or XML.
  • Client-Server Architecture: REST separates client and server concerns, allowing independent evolution.

REST APIs are simple, scalable, and widely supported, making them a popular choice for building web services and connecting different applications.
